### The Productivity Paradox: Where AI Actually Helps

**Clear wins:**
- Documentation generation/maintenance: 30‚Äì60% time savings, easy to validate
- Test scaffolding: high value, bounded scope, straightforward to verify
- Boilerplate & repetitive patterns: CRUD, config files, migration scripts
- New developer onboarding: AI context reduces senior engineer burden
- PR description summaries: underrated time saver

**Proceed with caution:**
- Security-sensitive code (crypto, auth, input validation) ‚Äî subtle AI errors common; always layer Snyk Code
- Complex domain logic (financial rules, medical data) ‚Äî context depth insufficient
- Debugging race conditions / distributed failures ‚Äî AI still weak at systemic async reasoning
