## LinkedIn PA Agent Cron & Fix (12:22–12:49 UTC)

**Trigger:** Cron job `linkedin-pa-agent-cron` executed (hourly UTC).

**Observation:** Agent completed but produced posts consistently below 300 words (e.g., 217–275 words). Root cause: dynamic query generation failing due to invalid `openclaw agent` command syntax in script.

**Issue details:**
- Script used: `openclaw agent --agent main --json <<<'{"messages":[...]}'` (incorrect)
- Correct pattern (as used by `meta-agent`, `mewchat-evolver`): `--message "$PROMPT"`
- This caused the dynamic query to be empty, falling back to static pool. Static queries may not retrieve rich content, leading to sparse posts.

**Fix applied:**
- Updated `agents/linkedin-pa-agent.sh` line generating `DYNAMIC_QUERY` to:
  ```bash
  DYNAMIC_QUERY=$(openclaw agent --agent main --message "$PROMPT" --json 2>/dev/null | jq -r '.response.content' 2>/dev/null || echo "")
  ```
- Verified syntax consistency with other agents.

**Expected outcome:** Next hourly run should generate a proper dynamic query, fetch more relevant sources, and produce posts meeting the 300-word target.

**Follow-up:** Monitor next few runs to confirm quality improvement and no runtime errors.
